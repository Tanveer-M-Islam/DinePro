Project Overview

DinePro is a modern, responsive Restaurant Management System built with Laravel. It provides a complete solution for restaurants to manage menus, reservations, customer reviews, and website settings through an intuitive admin panel.

The frontend offers an elegant, user-friendly interface for customers to browse the menu, book tables, and read reviews.

Features List

  • Dynamic Menu Management: Create categories and menu items with images, prices, and descriptions.
  • Table Reservation System: Customers can book tables online; admins can manage reservations.
  • Customer Reviews: Collect and display customer testimonials (with admin moderation).
  • Multi-Theme Support: Admin can switch between multiple pre-defined themes (Elegant Midnight, Sunset Glow, etc.).
  • SEO Optimization: Configurable SEO titles, keywords, and descriptions.
  • Responsive Design: Fully optimized for mobile, tablet, and desktop devices.
  • Admin Dashboard: Comprehensive control over all website content and settings.

Technology Stack

Component Technology
Backend Framework Laravel 11 (PHP 8.2+)
Database MySQL
Frontend Blade Templates, Tailwind CSS, Alpine.js

Installation Guide (Local Setup)

Prerequisites: ensure you have PHP >= 8.2, Composer, and MySQL installed on your machine.

1. Clone or Extract Project

Extract the project files to your local server directory (e.g., htdocs or www).

2. Install Dependencies

composer install

If you have node dependencies (for development):

npm install

3. Configure Environment

Copy the example environment file and create your own .env file:

cp .env.example .env

Open .env and configure your database settings:

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=dinepro
DB_USERNAME=root
DB_PASSWORD=

4. Generate Application Key

php artisan key:generate

5. Setup Database

Create a MySQL database named dinepro (or whatever you specified in .env). Then run migrations and seeders:

php artisan migrate --seed

The --seed flag populates the database with default admin user and initial settings.

6. Link Storage

php artisan storage:link

7. Serve Application

php artisan serve

Access the site at http://localhost:8000.

Shared Hosting Deployment Guide

1. Upload Files

Upload all project files to the root directory of your hosting account (e.g., above public_html). Let's call this folder dinepro_core.

2. Move Public Assets

Move the contents of the public folder to your domain's public directory (e.g., public_html).

3. Update index.php

Edit public_html/index.php to point to the correct paths:

require __DIR__.'/../dinepro_core/vendor/autoload.php';
$app = require_once __DIR__.'/../dinepro_core/bootstrap/app.php';

4. Set Permissions

Ensure the following directories are writable (755 or 775):

  • dinepro_core/storage
  • dinepro_core/bootstrap/cache

5. Configure .env

Update dinepro_core/.env with your production database credentials and set:

APP_ENV=production
APP_DEBUG=false
APP_URL=https://your-domain.com

6. Setup Database

Export your local database and import it via phpMyAdmin on your hosting server.

Free Render Deployment (Demo)

Recommended for Demos: This guide sets up a free deployment that resets the database on every deploy. ideal for showcasing the app.

1. Push to GitHub

Ensure your project is pushed to a GitHub repository.

2. Create Web Service on Render

  1. Go to Render Dashboard.
  2. Click New + -> Web Service.
  3. Connect your GitHub repository.

3. Configure Service

  • Name: dinepro-demo
  • Region: Any (e.g., Singapore)
  • Branch: main
  • Root Directory: (Leave empty)
  • Runtime: PHP
  • Build Command: ./deploy.sh
  • Start Command: php artisan serve --host=0.0.0.0 --port=$PORT

4. Environment Variables

Add the following environment variables in the Render dashboard:

  • APP_KEY: (Run php artisan key:generate --show locally and copy the value)
  • APP_DEBUG: true
  • APP_ENV: production
  • APP_URL: (Your Render URL, e.g., https://dinepro-demo.onrender.com)
  • DB_CONNECTION: sqlite
  • DB_DATABASE: /var/www/html/database/database.sqlite (Important: Use absolute path if possible, or relative)

Note: Since SQLite file is ephemeral on Render free tier, data will reset on redeploy. Perfect for demos.

5. Access Demo Admin

Once deployed, use these dummy credentials:

  • Login URL: /admin/login
  • Email: admin@example.com
  • Password: password

Theme Customization Guide

Switching Themes

Log in to the Admin Panel, go to Themes, and select a predefined theme (e.g., Sunset Glow, Elegant Midnight).

Modifying Themes

Theme configurations are stored in config/themes.php.

To change colors or fonts:

  1. Open config/themes.php.
  2. Locate the theme array (e.g., 'elegant_midnight').
  3. Update the hex codes in the 'colors' array.
'elegant_dark' => [
    'name' => 'Elegant Midnight',
    'colors' => [
        'primary' => '#fbbf24', // Change this hex code
        'background' => '#111827',
        // ...
    ],
]

CSS variables in the frontend are automatically generated from this configuration.

Public Routes
